Device simulation

IoT Device manager offers the possibility to simulate sending of measures from devices.

DeMa gives two way of simulation, standard simulation where user simulate his device with already predefined default values and custom simulation where user can manually configure attributes, its value range and how often these values will be sent.

The simulated values are shown (in any case) in real time in the table on the right of the page, as shown in the following picture:

These measures are shown also in case of real measures sent by the device.

As well as real measures, the simulated measures will follow the data flow within the Digital Enabler (harmonization, history, elaboration, visualization).

Standard simulation#

We have 3 different way to start the standard simulation.

The first one is to start it from the result list of filtering. Selecting the device (1) and clicking on "Fire a simulation" (2).

In this case it starts the standard simulation on all attributes of the selected devices. The red icon on the badge shows that a simulation is running on this device.

Opening the device details the user can click on the button "Device simulation". Also in this case it starts the standard simulation on all attributes of the current device.

In the "Attributes" section the user can decide to start the simulation for each single attribute. Using the left button (1) it uses the standard simulation, using the right button (2) it uses the custom simulation.

Custom simulation#

DeMa also offers the possibility of a custom simulation for the measures. On the "Attributes" section of the device details page, the users have to click on the "settings" icon (1) to open a popup where it is possible to configure some fields, based on the type (text, numbers, decimal, date, etc) (2)(3) of the attribute.

If the type is "text" the popup will be like the following picture:

After the name and the type of the attribute, you can see a section where the user can insert the words to simulate. The user can remove the default values ("text1","text2","text3") and add the words he/she wants.

At the bottom there is a slider to set the refresh interval (in seconds), in other word how often a new measurement must be simulated/sent.

If the type is "number", "binary" or "hex" the popup will be like the following picture:

In this case the user can define the range within which the random numbers will be generated, selecting the lower and upper limits.

In case of "decimal" type, the limits will be in decimal format:

In case of "boolean" and "datetime" type the use can only define the "simulation delay execution"

For boolean type the value will be obviously true/false:

For datetime type the value will be the current timestamp
