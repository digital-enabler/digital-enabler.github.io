Operator’s guide

Custom Operator​

The custom operator options allow you to create your operator based on your needs, click "create custom operator" to start building your own operator.

Once the custom operator gui is open you will see the following screen:

On your left you have the operator code, the instruction that will be executed by the operator, on the right you have the operator metadata. The code of the operator needs to be written in Javascript. The operator metadata is composed by the following fields:

  • Name: the name of the operator
  • Description: a description of the operator
  • Tags: a list of tags to help you find the operator in the operator list
  • Inputs: the list of the inputs of the operator
    • Name: the name of the input (name should not contains space or special characters)
    • Type: the type of the input
    • Description: a description of the input

Object​

Add properties​

Add some properties to an existing object given in input to the operator.

Input/output example

Connect your input to the operator, within the source tree, locate and select from the input one of the object. You can do this by clicking on the appropriate node within the source tree and drag-and-drop it into the source object field

Once your object is in the Source Object you can start add more property to it.

As you can see under the Source Object field we have two more section in the modal:

Within the properties section, you will find the newly added property for the object, accompanied by two labels. These labels can be populated with the Key and the Value corresponding to the object's new property.

Click the red button "ADD PROPERTY"

When you are done just click OK to close the modal.

If a property has been mistakenly added or contains an incorrect value, you will find an edit icon (represented by a a pencil) and a delete icon (represented by a trash bin) located to the right of the property. The edit icon can be utilized to modify the property, while the delete icon allows for its removal.

Key/Value Pair List​

This operator allows to convert an input object into an array of key-value pairs.

Input/Output examples

How to use it

Within the source tree, locate and select the input object that you want to convert into an array of key-value pairs. You can do this by clicking on the appropriate node within the source tree.

With the desired input object selected, drag and drop it from the source tree modal into the designated source object field within the operator's interface. This action associates the selected object with the conversion process.

After dropping the input object into the source object field, click the "OK" button to confirm your selection. The operator will then process the input and generate the output, which will be an array of key-value pairs.

Pick properties​

This operator allows to create a new object that contains only the selected properties from another object.

Input/Output examples

How to use it

After connecting your input into the operator you can drag-and-drop your object into the Source Object field.

Below the source object, you will find the "Properties" field and the "Property" section. Currently, the Properties field is empty because it will display all the individual properties you select from your input.

The Property field contains a list of properties within the current source object. Click on it to reveal a drop-down list and select the desired property that you wish to include in your new object. Afterward, click the red "ADD PROPERTY" button.

Observe that our property has successfully been included in our new object. If needed, you can remove an added property by utilizing the trash can icon located to the right of the property. Once you have finished, simply click the OK button to close the modal.

Rename properties​

This operator allows your to rename properties of your object given in input.

Input/Output examples

How to use it

Connect your input and drag-and-drop it into the source object field.

The properties field will contains all the properties that you will rename, to do that you need to click on the drop-down list below property and select one.

After that you want to insert some new name into the value field, and click the red button "add property"

Notice that below the properties, our property is now displayed with its updated name. In the event of a typographical error, you can still modify its name using the pen icon located on the right.

Additionally, the trash bin icon is also available on the right side, enabling you to delete the previously added property, similar to other operators.

In other words, when merging two data arrays, you establish a correspondence between a key field in the first array and a key field in the second array. If the values of these key fields match, then the data from the two arrays can be merged together

Sort by property​

Sorts the elements of Source array by the values of Property.

Input/output examples:

Connect your input to the operator, within the source tree, locate and select from the input the property that you want to sort. You can do this by clicking on the appropriate node within the source tree and drag-and-drop it into the source array field

String​

Change case​

Change text case (upper, lower or camel)

Input/Output examples:

How to use:

Connect your input to the operator, within the source tree, locate and select from the input the string that you want to change case. You can do this by clicking on the appropriate node within the source tree and drag-and-drop it into the string field

Lower: In this transformation, all the letters in your string will be converted to lowercase. For example, "HELLO" will become "hello".

Upper: This conversion will change all the letters in your string to uppercase. For instance, "hello" will become "HELLO".

Camel: By applying the camel case convention, the string provided as input will be modified. The first letter will be capitalized, and any subsequent words will have their first letter capitalized as well. For example, "hello" will be transformed into "Hello".

Format Date​

Format date to a specific format (ISO-8601, European, USA or custom).

Input/Output examples:

Drag-and-drop your input into the Datetime Expression, it can be a Unix Timestamp or a string compliant with ISO-8601.

Now it's the turn to select a Data format, this is a drop-down list and have the most popular Date format, you can even create a custom one.

If you choose the "Custom" option, you will notice an additional field in the modal called "Custom Format." This field allows you to enter a specific pattern for your desired date format.

When using a custom format, you can modify it by utilizing various options. For example:

  • YY represents the short-year
  • YYYY represents the full-year
  • MM represents the month
  • DD represents the day
  • hh represents the hours (1-2 digits)
  • mm represents the minutes
  • ss represents the seconds
  • ttt represents the milliseconds

By combining these options, you can create a custom format that suits your specific date formatting requirements.
